# Pulse Architecture — Module Responsibilities

This document describes the current module structure of Pulse, their
responsibilities, and how they interact in the execution pipeline.

---

## Execution Pipeline

```
source.pul
    │
    ▼
┌─────────┐     tokens      ┌─────────┐     AST         ┌──────────┐
│  Lexer  │ ──────────────► │ Parser  │ ──────────────► │ Resolver │
└─────────┘                 └─────────┘                 └──────────┘
                                                              │
                                                         resolved AST
                                                              │
                                                              ▼
                                                      ┌─────────────┐
                                                      │ Interpreter │
                                                      └─────────────┘
```

---

## 1. `src/tokens.py`

Defines the token system used by the lexer.

- `TokenType` — enum of all token categories (keywords, literals, operators, structural)
- `Token` — dataclass holding type, lexeme, literal value, line, and column
- `KEYWORDS` — mapping of keyword strings to their `TokenType`

---

## 2. `src/lexer.py`

Converts raw `.pul` source code into a flat list of `Token` objects.

- Scans source character by character
- Handles: identifiers, keywords, numbers, strings, f-strings, tensor literals, operators, punctuation, comments
- Manages indentation-based block structure — emits `INDENT` / `DEDENT` tokens automatically
- Raises `PulseLexError` on illegal characters, invalid escape sequences, unterminated strings, or bad indentation
- Output: `List[Token]`

---

## 3. `src/expressions.py` / `src/statements.py`

Defines all AST node classes, generated by `generate_ast.py`.

- `expressions.py` — all expression node types (`Binary`, `Unary`, `Call`, `Lambda`, `ListComp`, `Ternary`, `Tensor`, etc.)
- `statements.py` — all statement node types (`Function`, `Class`, `If`, `For`, `Match`, `Try`, `Import`, `Raise`, `Del`, etc.)
- Each node implements `accept(visitor)` for the visitor pattern
- `ExprVisitor` and `StmtVisitor` — abstract base classes defining the visitor interface

---

## 4. `generate_ast.py`

Code generation script that produces `expressions.py` and `statements.py`.

- Run once to regenerate AST files after adding new node types
- Defines `EXPRESSIONS` and `STATEMENTS` dicts mapping node names to their fields
- Outputs properly typed, self-documenting Python dataclasses

---

## 5. `src/parser.py`

Reads the token stream and produces an AST.

- Recursive-descent parser following Pulse's grammar
- Produces all expression and statement nodes
- Handles operator precedence, associativity, and chained comparisons
- Desugars augmented assignment (`x += 1` → `Assign(x, Binary(x, +, 1))`) during parsing
- Raises `PulseSyntaxError` with line, column, and context snippet on invalid syntax

---

## 6. `src/resolver.py`

Static scope analysis pass that runs between parsing and interpretation.

- Walks the AST and resolves variable references to their lexical scope depth
- Detects errors before runtime: use before definition, return outside function, break/continue outside loop
- Stores resolution distances in `interpreter.locals` for fast variable lookup
- Does not execute any code — read-only analysis pass

---

## 7. `src/interpreter.py`

Tree-walk interpreter that executes the resolved AST.

- Implements `ExprVisitor` and `StmtVisitor` — one `visit_*` method per node type
- Evaluates all expressions and executes all statements
- Manages the current `Environment` for variable storage and lexical scoping
- Handles function calls, class instantiation, method dispatch, closures
- Implements the `BuiltinFunction` fast-path — C-backed and stdlib functions bypass the full call pipeline
- Built-in functions defined directly on the interpreter: `print`, `input`, `str`, `int`, `float`, `bool`, `type`, `abs`, `pow`, `min`, `max`, `len`, `range`, `round`, `sum`, `any`, `all`, `enumerate`, `zip`
- Raises `PulseRuntimeException` on runtime errors

---

## 8. `src/environment.py`

Manages variable storage and lexical scoping.

- `Environment` — stores variable bindings in a flat dict with an `enclosing` pointer
- Supports: `define`, `get`, `assign`, `delete`, `get_at`, `assign_at` (for resolver-resolved lookups)
- Scope chain: local → enclosing → ... → global
- Each function call, block, and class body gets its own `Environment`

---

## 9. `src/function.py`

Defines all callable types in the Pulse runtime.

- `PulseFunction` — wraps a user-defined function with its closure environment; handles default parameters, `*args`, and keyword arguments
- `PulseNativeFunction` — wraps a Python callable for general native dispatch
- `PulseNativeMethod` — wraps a Python callable for method dispatch on built-in types
- `PulseLambda` — anonymous function with a single expression body
- `BuiltinFunction` — fast-path callable for C-backed and stdlib functions; bypasses environment setup and call frame overhead

---

## 10. `src/values.py`

Defines all runtime value types.

- `PulseNumber` — wraps `int` or `float`
- `PulseString` — wraps `str`
- `PulseBoolean` — wraps `bool`
- `PulseNull` — represents `null`
- `PulseList` — wraps `List[PulseValue]`
- `PulseDict` — wraps `Dict[PulseValue, PulseValue]`
- `PulseRange` — wraps a lazy integer range
- `PulseTensor` — wraps a NumPy `ndarray`
- `PulseModule` — holds a named dict of exported members
- `PulseModel` — wraps a scikit-learn model with Pulse methods
- `PulseNamespace` — lightweight named member container

---

## 11. `src/runtime.py`

Defines control-flow signals and the class/instance system.

- `BreakException`, `ContinueException`, `ReturnException` — used internally to unwind the call stack for loop control and function returns
- `PulseException`, `PulseRuntimeException`, `PulseValueError`, `PulseTypeError` — user-visible exception hierarchy
- `PulseClass` — runtime class object with methods, class variables, and base class chain
- `PulseInstance` — runtime instance with field dict and class reference

---

## 12. `src/error.py`

Centralised error types and formatted error reporting.

- `PulseError` — base error class
- `PulseLexError` — lexer errors with line, column, and source context
- `PulseSyntaxError` — parser errors with line, column, and source context
- `PulseSemanticError` — resolver errors
- `PulseRuntimeError` — runtime errors with stack trace support (`push_stack` / `pop_stack`)
- `report_error` — prints a formatted, human-readable error to stderr

---

## 13. `src/native.py`

Unified ctypes bindings for all C native extensions.

- Loads `bin/pulse_loader.dll` and `bin/pulse_math.dll` at import time
- `find_module(name, search_dirs)` — resolves a module name to a file path using C
- `read_file(path)` — reads a file into a Python string using C, with safe heap management
- `make_c_math_functions(interp_raise)` — returns a dict of C-backed math callables

---

## 14. `native/`

C source files for native performance extensions.

- `pulse_loader.c` — file path resolution and file reading via `stat` / `fopen` / `fread`
- `pulse_math.c` — math functions (`sqrt`, `sin`, `cos`, `log`, etc.) with C-level validation
- `build.bat` — builds all C sources into `bin/` on Windows

---

---

## 16. `pulse.py`

Main entry point for the Pulse language.

- Parses CLI arguments using `argparse`
- `py pulse.py file.pul` — runs a Pulse program through the full pipeline
- `py pulse.py` — starts the interactive REPL
- `py pulse.py file.pul --time` — runs with per-stage pipeline timing (lex, parse, resolve, interpret)
- `py pulse.py --info` — prints the full language reference guide
- REPL supports multi-line input, auto-indent, error recovery, and arrow-key history

## Summary

| Layer        | Files                                                                        | Role                              |
| ------------ | ---------------------------------------------------------------------------- | --------------------------------- |
| Tokenization | `tokens.py`, `lexer.py`                                                      | Source → tokens                   |
| Parsing      | `parser.py`, `expressions.py`, `statements.py`, `generate_ast.py`            | Tokens → AST                      |
| Analysis     | `resolver.py`                                                                | AST → resolved AST                |
| Execution    | `interpreter.py`, `environment.py`, `runtime.py`, `function.py`, `values.py` | AST → output                      |
| Native       | `native.py`, `native/*.c`, `build.bat`                                       | C-backed performance extensions   |
| Stdlib       | `src/stdlib/*.py`                                                            | Built-in modules                  |
| Errors       | `error.py`                                                                   | Unified error reporting           |
| Entry point  | `pulse.py`                                                                   | CLI, REPL, pipeline orchestration |
